# 实现方案与架构决策(B站优先)

记录流明 AI VTuber 从"对标调研"收敛出的关键技术决策与 why,防止后续回滚到已被否决的方向。结论基于:Neuro-sama 等成功案例的实证、Open-LLM-VTuber / kimjammer-Neuro 的代码实测、以及 B站/抖音 的推流与合规政策核实。

## 决策一:首发平台 = B站,不是抖音

README 原定推流抖音;调研后改为 **B站 优先验证**。四条证据全部指向 B站。

| 维度 | B站 | 抖音 |
|---|---|---|
| 弹幕接入 | 开放(WebSocket / blivedm),开源已支持 | 无官方公开 API,需逆向,不稳 |
| 纯 AI 无人合规 | 基本可行(仅需 AI 标注),整治不针对娱乐 VTuber | 明确违规,要求真人实时值守 |
| 平台文化 | VTuber 老家,有对口观众与分区 | 带货为主,陪聊 AI 无成功先例 |
| 已验证案例 | Neuro-sama、AI-YinMei(吟美)均在 B站 | 无 |
| 变现天花板 | 打赏(舰长/SC/电池)真实但盘子小 | 流量与钱大得多 |

**Why 防回滚**:抖音的大流量绑定了三道墙——带货式合规枷锁、无弹幕 API、纯 AI 无人直播违规——三者都正面顶撞"纯 AI 陪聊 + 触发打赏"的 MVP 命题。抖音留作 B站 验证成功后、愿意额外承担真人值守+弹幕逆向+带货合规时的二期目标。

## 决策二:形态 = 2D Live2D 起步,UE5 后置为画面升级项

否决 README 原定的"UE5 + Pixel Streaming + 云 GPU"作为 MVP 前置。

**Why**:产品价值在对话闭环(延迟 + 人格 + 触发节奏),不在渲染保真度——Neuro-sama 用简单 2D 模型成为 Twitch 订阅第一即为实证。UE5 路线把复杂度压在"渲染"上,额外背上云 GPU 常驻成本、面部 rig、Pixel Streaming 稳定性、hero 美术关键路径,而这些都不直接产出要验的 liveness。2D/VRM 上"LLM 意图→动作/表情"的导演层逻辑同构,验证成本低一个数量级。UE5 当成画面升级项,不作 MVP 依赖。

## 决策三:技术栈选择

| 层 | 选择 | Why |
|---|---|---|
| 编排框架 | Open-LLM-VTuber(pin v1 自维 fork) | 已接 B站 弹幕、火山豆包零代码、TTS 插件化、中文文档 |
| 弹幕接入 | blivedm(框架自带 LivePlatformInterface) | B站 弹幕开放,填 room_id + SESSDATA 即用 |
| LLM | 火山豆包(Ark,OpenAI 兼容) | 区域内低延迟、中文好、便宜、零代码接入 |
| TTS | 火山流式 + 声音复刻 / MiniMax | 流式低延迟 + 统一音色;MiniMax 角色音情感强且框架内置 |
| 形象 | Live2D Cubism(自购/委托,商用授权) | B站 VTuber 标准、轻、proven |
| 采集推流 | OBS 浏览器源 → B站 rtmp | B站 直播中心/直播姬给 rtmp,标准链路 |
| 部署 | 单台带 GPU 的 PC + 托管 API | 2D + 托管 LLM/TTS,MVP 无需云 GPU |

**关键认知**:LLM 人格主要由 system prompt + 角色设定 + 少量微调决定,换基座模型解决不了"无聊"。

## 决策四:自建 vs 借开源 的边界

判据(满足任一才自建,否则借开源):①是差异化;②开源没有且承重/担责;③开源设计顶撞核心 KPI。

**自建(按优先级)**:

| 模块 | 判据 | Why 自建更好 |
|---|---|---|
| 导演层 + "何时说话"触发节奏 | ① | 核心增量;开源触发逻辑是通用玩具,越 specific 越赢 |
| 角色人格 + 一致性记忆 | ① | 护城河;开源记忆通用/残缺(Open-LLM-VTuber 长期记忆已下线) |
| 内容安全双向护栏 | ② | 开源为零且是法律担责项,不可外包 |
| 流式 TTS 低延迟管线 | ③ | 开源是整句文件式契约,顶撞"超速回应"KPI,需换内核 |
| 推流/采集无人化稳定性 | ② | 开源靠人肉 OBS;24h 长稳需 headless 渲染→编码→rtmp+监控(MVP 后) |

**借开源(造了即浪费)**:B站弹幕协议(blivedm)、Live2D 渲染+音量驱动口型、LLM/TTS provider 适配层、弹幕背压队列(ProxyMessageQueue)、ASR/VAD。

## 决策五:合规约束(B站,全国性国标)

- 画面标注"虚拟主播/AI生成"(≥1/10 区域);开播前 10 秒语音+字幕表明 AI 身份。
- 双向敏感词过滤:入(弹幕)+出(LLM 回复)各一道,避免被弹幕诱导导致封播;预算够再加云内容安全 API 语义审核。
- **Why**:国标《人工智能生成合成内容标识办法》对所有平台适用;B站 整治压力虽小于带货平台,但敏感内容封播风险与账号主体担责不可裸奔。

## 已知选型风险

- **Open-LLM-VTuber v1 已冻结、官方将重写 v2 且不兼容**:pin v1 自维护;预期导演层长大到与通用框架分叉时,把编排内核收归自建。
- **框架自带 Live2D 样本模型商用受限**(企业年销 >1000 万日元须 Live2D 书面授权,协作角色禁商用):正式上线必须换有商用授权的模型。
- **抖音弹幕源无官方 API**:若二期上抖音,弹幕获取有稳定性与合规风险。
