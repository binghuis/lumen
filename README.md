# 流明 · AI VTuber

## 产品形态

- **定位**:AI-native 陪伴/互动型虚拟主播(机智聊天 + 弹幕超速回应 + 定制人格 + 自然动作反馈 + 面部表情口型)。\
- **不是**:3D 唱跳偶像、电商口播数字人。

## 技术架构

```
弹幕/礼物事件
   │
   ▼
豆包 LLM〔火山·云〕→ 对话内容 + 动作意图标签
   ├─→ 火山 TTS〔云·声音复刻统一音色〕→ 口型驱动(Audio2Face / audio→viseme)→ 风格化角色面部
   └─→ UE5〔云 GPU〕:意图标签触发动作 montage + Motion Matching 待机混合
                          │
              风格化角色 + 场景渲染(Lumen / PBR)
                          │
                  Pixel Streaming → 推流抖音
                          ▲
          美术资产(风格化角色 / 场景 / 着色,离线产出)← 关键路径
```

**核心增量** = "LLM 意图 → UE5 动作"的导演层。难的不是生成动作,是 LLM 在对的时机从动作库里选对反馈、再和待机平滑混合——前者你的弹幕闭环本来就知道时机,后者是引擎标准能力。

## 技术栈选型

| 环节 | 选型 | 成熟度 |
| --- | --- | --- |
| 对话闭环(读弹幕+语音回应) | 参考 kimjammer/Neuro 栈;LLM = 火山豆包(云 API),语音 = 火山流式 TTS(云) | ✅ 架构已落地;云 API 免常驻 GPU,弹幕高并发更省 |
| 渲染 / 部署 | UE5 + Pixel Streaming,跑云 GPU,推流抖音 | ⚠️ 无本地显卡 → 必走云 GPU;MVP 按小时租,24h 量产成本月级几千元(估算) |
| 角色 / 画风 | 自定义风格化 CG 高模(非 MetaHuman、非轻量 VTuber 皮) | ⚠️ 美术 = 关键路径,来源待定;MVP 先用现成/占位模型验导演层 |
| 面部口型 / 表情 | Audio2Face 或等效 audio→viseme,驱动角色 ARKit/52-blendshape 面部 rig | ⚠️ 需角色绑面部 rig;随渲染上云 GPU |
| 音色统一 | 火山 TTS 声音复刻直接产出统一音色,去掉 Seed-VC 变声层 | ✅ 云 TTS 已统一音色;本地无 GPU、Seed-VC 选项已无,确认去除 |
| 自然动作反馈(核心增量) | UE5 触发式动作库 + Motion Matching 待机(骨干);生成式 co-speech gesture 留作后续升级 | ✅ 引擎标准能力 / ⚠️ 升级项待验证 |
| 才艺 | 礼物触发预渲染片段,不做实时 AI 唱跳 | ✅ 绕开瓶颈 |

## 开放问题(落地前要解)

- **抖音推流地址**:整条管线在云上直推抖音时,推流 URL+key 怎么稳定获取(直播伴侣 / 开放平台)。
- **弹幕读取**:从抖音侧稳定拿到弹幕 / 礼物事件流,喂给导演层。

## MVP 验证范围

**只验一件事**:纯 AI 无人能不能跑出"弹幕来了 → 对的动作反馈 + 对的话 + 面部同步、且不僵硬"的 liveness,以及能不能触发打赏。

MVP 阶段**先借后造**:现成 / 占位风格化模型 + 按小时租云 GPU,先把导演层验掉,不等 hero 美术。
